# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1.0 | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in OpenCut, please report it responsibly.

**Do not** open a public issue. Instead, send an email to depontefede@gmail.com with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

We aim to respond within 48 hours and will work with you to verify and address the issue promptly.

## Security Best Practices

When using OpenCut:

- Do not pass untrusted user input directly to CLI commands
- Validate all file paths before processing
- Keep Remotion and other dependencies up to date
- Run renders in isolated environments when processing untrusted projects
